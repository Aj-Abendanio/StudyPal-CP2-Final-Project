Module 10: Parallel and Distributed Computing
1. Introduction
As the physical limits of CPU clock speeds are reached, modern computing relies on doing multiple things at once. Parallel and Distributed Computing explores how to break large problems into smaller parts that can be executed simultaneously. Parallel computing typically happens on a single machine with multiple processors (multi-core), while distributed computing involves multiple autonomous computers linked via a network (clusters or cloud) working toward a common goal.
2. Key Concepts

- Concurrency: The ability of different parts or units of a program to be executed out-of-order or in partial order, without affecting the final outcome. 
- Parallelism: The simultaneous execution of multiple tasks. While concurrency is about dealing with lots of things at once, parallelism is about doing lots of things at once. 
- Nodes: Individual computers in a distributed system that communicate with each other. 
- Shared Memory: A model where multiple processors can access the same memory space (common in multi-core PCs). 
- Distributed Memory: A model where each processor has its own private memory and must communicate via "Message Passing." 
- Scalability: The ability of a system to handle a growing amount of work by adding resources (Vertical scaling = bigger CPU; Horizontal scaling = more computers). 
- Fault Tolerance: The ability of a distributed system to continue operating even if one or more nodes fail.
3. Examples

- Multi-core Processing: When you render a 4K video, your software splits the frames across all 8 or 16 cores of your CPU. This is Parallel Computing . 
- Google Search: When you type a query, the request isn't handled by one computer. It is distributed across thousands of servers worldwide. This is Distributed Computing . 
- SETI@home: A famous project where millions of home computers worldwide collaborated via the internet to analyze radio signals for signs of extraterrestrial life. 
- Race Conditions: A common bug in parallel programming where two "threads" try to update the same variable at the exact same time, leading to unpredictable results.
4. Summary

- Parallelism focuses on speed and performance within a single system. 
- Distributed Systems focus on reliability, scalability, and resource sharing across networks. 
- Amdahl's Law is used to predict the maximum improvement to a system when only part of it is improved.
