Module 17: Blockchain Technology
1. Introduction
Blockchain is a distributed, decentralized ledger technology that records transactions across many computers in such a way that the registered transactions cannot be altered retroactively. While most famously known as the foundation for cryptocurrencies like Bitcoin, blockchain has broader applications in supply chain management, voting systems, and smart contracts. It solves the "Double Spend" problem and removes the need for a central trusted authority (like a bank) by using cryptographic proof.
2. Key Concepts

- Block: A container data structure that aggregates transactions for inclusion in the public ledger. It consists of a header and a list of transactions. 
- Distributed Ledger: A database that is shared and synchronized across multiple sites, institutions, or geographies, accessible by multiple people. 
- Decentralization: The transfer of control and decision-making from a centralized entity (individual, organization, or group) to a distributed network. 
- Hash Pointer: A pointer to where data is stored together with a cryptographic hash of the value of that data at some point in time. This makes the chain "immutable." 
- Consensus Mechanism: A fault-tolerant mechanism used in computer and blockchain systems to achieve the necessary agreement on a single data value or a single state of the network among distributed processes (e.g., Proof of Work, Proof of Stake). 
- Smart Contract: Self-executing contracts with the terms of the agreement between buyer and seller being directly written into lines of code.
3. Examples

- Immutability: If a hacker tries to change the data in "Block 2," the hash of Block 2 changes. Since "Block 3" contains the old hash of Block 2, the link is broken, and the network rejects the change. 
- Smart Contracts (Ethereum): Imagine a vending machine. If you put in the money and select a snack, the snack is released. A smart contract does this with digital assets (e.g., "If $X$ sends 5 tokens, then release the digital deed to the house"). 
- Supply Chain: A company can use blockchain to track a shipment of medicine. Every stop from the factory to the pharmacy is recorded on the ledger, ensuring the medicine wasn't swapped for a counterfeit version. 
- Proof of Work (Mining): Computers compete to solve a complex mathematical puzzle. The first one to solve it gets to add the next block to the chain and receives a reward.
4. Summary

- Blockchain provides transparency and security without a central authority. 
- The data is immutable ; once written, it is nearly impossible to change without alerting the entire network. 
- Consensus algorithms ensure that all nodes in the network agree on the current state of the ledger.
