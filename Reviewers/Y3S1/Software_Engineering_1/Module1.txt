Module 12: Software Engineering 1
1. Introduction
Software Engineering is the systematic, disciplined, and quantifiable approach to the development, operation, and maintenance of software. Unlike simple "coding," software engineering treats software development as an engineering problem that requires careful planning, risk management, and quality assurance. This module focuses on the Software Development Life Cycle (SDLC) and the methodologies used by teams to deliver reliable software on time and within budget.
2. Key Concepts

- SDLC (Software Development Life Cycle): The structured process used by the software industry to design, develop, and test high-quality software. 
- Waterfall Model: A linear, sequential approach where each phase (Requirements, Design, Implementation, etc.) must be completed before the next begins. 
- Agile Methodology: An iterative and incremental approach that focuses on flexibility, continuous delivery, and team collaboration (e.g., Scrum, Kanban). 
- Requirements Engineering: The process of defining, documenting, and maintaining the requirements of what the system must do. 
- Sprint: A set period of time (usually 2–4 weeks) during which specific work must be completed and made ready for review in Agile/Scrum. 
- Quality Assurance (QA): The process of ensuring that the software product meets the specified requirements and standards. 
- MVP (Minimum Viable Product): A version of a product with just enough features to be usable by early customers who can then provide feedback for future development.
3. Examples

- Waterfall vs. Agile: If you are building software for a NASA space probe, you might use Waterfall because the requirements cannot change once it is launched. If you are building a new social media app, you use Agile to release features quickly and adapt to user trends. 
- Functional vs. Non-Functional Requirements: 
  - Functional: "The user must be able to reset their password via email." 
  - Non-Functional: "The system must load the login page in under 2 seconds." 
- Scrum Roles: In a classroom project, the Product Owner decides what needs to be built, the Scrum Master ensures the team follows the process, and the Developers build the code. 
- Unit Testing: A software engineer writes a small script to test if a single "Login" function correctly rejects a wrong password before integrating it into the main app.
4. Summary

- Software Engineering focuses on scalability, maintainability, and reliability . 
- The SDLC provides the roadmap for a project from the initial idea to the final product. 
- Agile is the industry standard for most modern web and mobile projects due to its ability to handle changing requirements.
