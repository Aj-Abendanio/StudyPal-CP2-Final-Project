Module 4: Object-Oriented Programming (OOP)
1. Introduction
Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects," which can contain data and code. Data is represented in the form of fields (attributes), and code is represented in the form of procedures (methods). This approach allows developers to mirror real-world systems, making code more modular, reusable, and easier to maintain. Instead of writing a long list of instructions, OOP encourages organizing code into logical units.
2. Key Concepts

- Class: A blueprint or template for creating objects. It defines the attributes and behaviors that the objects will have. 
- Object: An instance of a class. If "Car" is the class, then "Your Blue Sedan" is the object. 
- Encapsulation: The practice of bundling data and methods into a single unit (the class) and restricting direct access to some of the object's components. This is often achieved using access modifiers like private or public. 
- Inheritance: A mechanism where one class (subclass) acquires the properties and behaviors of another class (superclass). This promotes code reusability. 
- Polymorphism: The ability of different classes to be treated as instances of the same class through inheritance. It literally means "many forms," allowing one method name to behave differently depending on the object. 
- Abstraction: The process of hiding complex implementation details and showing only the essential features of an object.
3. Examples

- Class vs. Object: 
  - Class: Character (has attributes like health and mana). 
  - Object: Warrior, Mage, or Archer. Each has its own specific health and mana values. 
- Inheritance: In a game like Minecraft , you might have a base class called Mob. Both Zombie and Creeper inherit from Mob. They both have a position and health, but the Creeper has a unique explode() method. 
- Encapsulation: Think of an ATM. You can use the interface to "Withdraw Money," but you cannot see or change the internal code that calculates the balance directly. The data is hidden for safety. 
- Polymorphism: A Shape class has a method draw(). When you call draw() on a Circle object, it draws a circle. When you call the same draw() on a Square object, it draws a square.
4. Summary

- OOP is built on Classes (blueprints) and Objects (instances). 
- The Four Pillars of OOP are Encapsulation, Inheritance, Polymorphism, and Abstraction. 
- Inheritance allows us to create a hierarchy and reuse code. 
- Encapsulation protects data from unintended interference or misuse.
