Module 5: Information Management & Relational Databases
1. Introduction
Information Management is the lifecycle of collecting, storing, managing, and maintaining data to ensure it is accurate, accessible, and secure. In Computer Science, this most commonly involves Relational Database Management Systems (RDBMS) . Instead of storing data in disorganized text files, we use structured tables and relationships to handle massive amounts of information efficiently, which is the backbone of almost every modern web application, bank, and social media platform.
2. Key Concepts

- Database (DB): A structured collection of data stored electronically. 
- Relational Database: A type of database that stores data in tables (relations) which are linked based on data common to each. 
- Schema: The "skeleton" or formal structure of the database, defining the tables, fields, and relationships. 
- Primary Key (PK): A unique identifier for a record in a table (e.g., a Student ID). It cannot be null and must be unique. 
- Foreign Key (FK): A field in one table that links to the Primary Key of another table, creating a relationship between them. 
- Normalization: The process of organizing data to reduce redundancy and improve data integrity (e.g., ensuring a customer's address is only stored in one place). 
- SQL (Structured Query Language): The standard language used to communicate with and manipulate relational databases.
3. Examples

- Entity vs. Attribute: If you are building a system for a university, an Entity would be Student. The Attributes would be FirstName, LastName, and Email. 
- Primary Key: In a Users table, the user_id (usually an auto-incrementing integer like 1, 2, 3...) is the Primary Key. No two users can have the same ID. 
- Foreign Key Relationship: 
  - Table Courses has a PK course_id. 
  - Table Enrollments has an FK course_id. 
  - This links a specific student's enrollment record to the details of the course they are taking. 
- SQL Query: To find all students with the last name "Tababan," you would use: SELECT
- FROM Students WHERE LastName = 'Tababan';
4. Summary

- Information Management ensures data is consistent, persistent, and secure . 
- Relational Databases use tables, rows (records), and columns (fields) to organize information. 
- Keys (Primary and Foreign) are the tools used to maintain unique records and connect different tables.
