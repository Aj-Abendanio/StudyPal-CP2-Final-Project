Module 3: Computer Systems & Architecture
1. Introduction
Computer Systems and Architecture explores the inner workings of a computer, acting as the bridge between software and hardware. While a programmer writes high-level code, the architecture determines how that code is translated into electrical signals and processed by the CPU. Understanding this "under-the-hood" mechanics is vital for optimizing performance, managing memory, and understanding how a machine actually executes logic.
2. Key Concepts

- Von Neumann Architecture: The standard design for most computers, consisting of a Central Processing Unit (CPU), memory, and Input/Output (I/O) devices, where both data and instructions are stored in the same memory. 
- Central Processing Unit (CPU): The "brain" of the computer, responsible for executing instructions. It consists of the ALU (Arithmetic Logic Unit) and the CU (Control Unit). 
- Instruction Cycle (Fetch-Decode-Execute): The continuous process by which the CPU retrieves a program instruction from memory, determines what the instruction is, and carries it out. 
- Registers: Small, high-speed storage locations within the CPU used to hold immediate data and instructions. 
- Memory Hierarchy: The organization of storage by speed and cost, ranging from the fastest ( Registers and Cache ) to the slowest ( Main Memory and Secondary Storage ). 
- Bus: A communication system that transfers data between different components inside a computer (e.g., Data Bus, Address Bus).
3. Examples

- The Fetch-Execute Cycle: When you click "Save" in a document, the CPU fetches the save command, decodes it into binary, and executes the instruction by sending the data through the Data Bus to your SSD. 
- ALU Operations: If your code says x = 5 + 3, the Control Unit identifies the addition operation and sends the numbers to the Arithmetic Logic Unit (ALU) to calculate the result (8). 
- Cache Memory: Think of RAM as a large bookshelf and the Cache as your desk. You keep the books you are currently reading on your desk because it is much faster to reach them there than walking to the bookshelf every time. 
- Input/Output (I/O): When you type on a mechanical keyboard, the I/O controller sends an interrupt signal to the CPU, which then processes the keystroke and displays the character on your monitor.
4. Summary

- Architecture defines the physical and logical components of a computer system. 
- The Von Neumann model is the foundation for modern computing, using a shared memory for data and instructions. 
- The Instruction Cycle is the heartbeat of the CPU, ensuring every line of code is processed in sequence.
