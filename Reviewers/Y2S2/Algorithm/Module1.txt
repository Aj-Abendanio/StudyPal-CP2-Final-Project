Module 6: Advanced Algorithms & Efficiency
1. Introduction
While "Data Structures" (Module 1) focus on how we store data, Algorithms focus on the most efficient way to process it. In computer science, an algorithm is not just a solution—it is a specialized tool designed to solve a specific problem under certain constraints. As you move into more advanced coursework, the focus shifts from simply "making it work" to "making it scale," ensuring your code can handle millions of data points without crashing or slowing down.
2. Key Concepts

- Divide and Conquer: An algorithmic paradigm that breaks a problem into smaller sub-problems, solves them, and combines the results (e.g., Merge Sort). 
- Sorting Algorithms: Procedures used to rearrange elements in a specific order (Numerical or Lexicographical). 
- Searching Algorithms: Methods for finding a specific item within a data structure (e.g., Binary Search). 
- Recursion: A technique where a function calls itself to solve smaller instances of the same problem. 
- Greedy Algorithm: An approach that makes the locally optimal choice at each step with the hope of finding the global optimum. 
- Heuristic: A "rule of thumb" or shortcut used to find a "good enough" solution when an exact optimal solution is too slow to calculate.
3. Examples

- Binary Search ($O(\log n)$): Instead of checking every item (Linear Search), Binary Search cuts the search area in half each time. 
  - Example: Finding a word in a physical dictionary. You open to the middle, see if your word is before or after that page, and repeat. 
- Merge Sort: A Divide and Conquer algorithm. It splits an array into halves, sorts them recursively, and then merges them back together. 
  - Example: Organizing a deck of cards by splitting it into small piles, sorting the piles, and then zipping them back together in order. 
- Bubble Sort ($O(n^2)$): A simple but inefficient sorting algorithm that repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order. 
- Fibonacci Sequence (Recursion): Calculating the $n$-th number by adding the two previous numbers ($F(n) = F(n-1) + F(n-2)$).
4. Summary

- Efficiency is the core of algorithm design; we aim for lower Big O values. 
- Binary Search is significantly faster than Linear Search but requires the data to be pre-sorted . 
- Divide and Conquer is a powerful strategy for breaking down complex problems.
