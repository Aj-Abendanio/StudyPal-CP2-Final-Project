Module 11: Mobile Programming
1. Introduction
Mobile Programming focuses on developing software applications specifically designed for handheld devices, such as smartphones and tablets. Unlike desktop development, mobile programming must account for unique constraints: limited battery life, varying screen sizes, touch-based input, and intermittent internet connectivity. This module covers the two primary approaches— Native and Cross-Platform development—and the fundamental lifecycle of a mobile application.
2. Key Concepts

- Native Development: Building apps for a specific platform using its official language (e.g., Swift for iOS, Kotlin or Java for Android). This offers the best performance and access to hardware. 
- Cross-Platform Development: Writing code once that runs on both iOS and Android using frameworks like Flutter (Dart) or React Native (JavaScript). 
- Activity / ViewController: The fundamental building block of a mobile UI, representing a single screen that a user interacts with. 
- App Lifecycle: The series of states an app goes through from the moment it is launched until it is terminated (e.g., Started, Resumed, Paused, Stopped ). 
- Responsive Layout: A design approach that ensures the UI adapts gracefully to different screen resolutions and orientations (Portrait vs. Landscape). 
- API Integration: Using Web Services (usually via JSON) to fetch data from the internet, allowing the mobile app to communicate with a central server.
3. Examples

- Lifecycle Management: When you are playing a mobile game and receive a phone call, the OS moves the game to the "Paused" state to save battery and memory. When you hang up, the app moves back to "Resumed" . 
- Intent/Navigation: In an Android app, if you click a "Share" button, the app sends an Intent to the OS to open another app (like Gmail or WhatsApp) to handle the data. 
- Local Storage: Apps like Spotify use Caching to store music data locally on your phone so you can listen even when you have no signal. 
- Sensors: Mobile programming allows you to access hardware directly, such as the Accelerometer (to detect if the phone is tilted) or GPS (to show your location on a map).
4. Summary

- Mobile development requires a focus on User Experience (UX) and efficient Resource Management . 
- Native apps offer high performance; Cross-platform apps offer faster development cycles for multiple devices. 
- Understanding the App Lifecycle is crucial to prevent memory leaks and ensure a smooth user experience.
