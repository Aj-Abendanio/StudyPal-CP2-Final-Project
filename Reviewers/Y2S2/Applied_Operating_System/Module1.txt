Module 7: Applied Operating Systems
1. Introduction
An Operating System (OS) is the most essential piece of software on a computer. It acts as an intermediary between the user, the application software, and the computer hardware. Without an OS, you would have to write code to manually control the CPU, move the read/write head of a hard drive, and manage every bit of RAM yourself. This module focuses on how the OS manages resources, schedules tasks, and ensures that multiple programs can run simultaneously without crashing the system.
2. Key Concepts

- Kernel: The core part of the OS that stays in memory and manages hardware communication, memory allocation, and process scheduling. 
- Process: A program in execution. It is an active entity with a program counter, registers, and variables. 
- Thread: The smallest unit of execution within a process. A single process (like a web browser) can have multiple threads (one for loading images, one for playing audio). 
- Process Scheduling: The method by which the OS decides which process runs on the CPU at any given time (e.g., Round Robin, First-Come-First-Served). 
- Deadlock: A state where two or more processes are unable to proceed because each is waiting for the other to release a resource. 
- Virtual Memory: A technique that allows the execution of processes that are larger than the physical RAM by using a portion of the hard drive as temporary memory. 
- System Call: The programmatic way in which a computer program requests a service from the kernel of the operating system.
3. Examples

- Multitasking: When you listen to Spotify while coding in VS Code, the OS is rapidly switching the CPU between these two processes . This happens so fast that it appears they are running at the same time. 
- Context Switching: When the CPU stops working on one process to start another, it must save the state of the first process. This "bookmarking" is called a context switch. 
- The "Deadlock" Scenario: Imagine Process A holds a Printer and needs a Scanner. Simultaneously, Process B holds the Scanner and needs the Printer. Neither can finish; they are in a Deadlock . 
- Paging (Virtual Memory): If you open 50 tabs in Chrome and run out of RAM, the OS moves the data from the "least used" tabs onto your SSD to keep the system running.
4. Summary

- The Kernel is the heart of the OS, managing the interaction between software and hardware. 
- Processes are the active programs, and Threads are the smaller tasks within them. 
- Scheduling Algorithms ensure the CPU is used efficiently and fairly.
